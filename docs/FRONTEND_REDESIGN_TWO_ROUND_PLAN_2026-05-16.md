# LocalCommandCenter 前端二轮细化规划

日期：2026-05-16  
适用范围：`local-gateway/frontend`  
规划目标：在不虚构业务内容的前提下，对现有前端做一次成体系的视觉与信息架构升级。每个板块都要有独立的“呈现装置”，但整体必须属于同一个世界观，避免现在这种“有主题词，但仍像普通后台”的状态。

---

## 0. 现状判断

### 一句话问题定义

当前界面已经从普通后台换成了“纸面作战室”语汇，但多数页面仍然主要靠标题文案和卡片命名在传达气质，真正的版式骨架、信息层级、组件形态、交互路径还没有完成结构性转译。

### 当前前端基线

- 全局基调已经偏纸张、木面、作战板。
- `Dashboard / Calendar / Download / Habits / Notes / Sandbox / Settings / Sync / Workflows` 都已有主题口号，但很多页面仍是“标准表单 + 标准卡片 + 少量修辞文本”。
- `AiChat` 是当前复杂度最高、信息密度最高的页面，已经出现“任务卡 / 规划面板 / 轮次 / 预览”雏形，但整体舞台感不够集中。
- `Tasks` 的 `WeekView` 仍以传统时间表为主，板块个性不够强。
- 侧边栏与顶栏还承担的是普通后台导航职能，没有变成世界观的一部分。

### 主要缺口

- 缺一个更强的总主题母体。
- 缺每个页面独有的“信息容器”。
- 缺全局一致的空间语法。
- 缺把真实业务字段转译成更有记忆点的版式方法。
- 缺一套从首页到 AI 的连贯动线。

---

## 1. 设计方向

### Context

这是一个个人本地指挥中心，不是企业 SaaS 控制台。它既有任务、笔记、时间、同步、下载、沙盒，又有 AI 规划和工作流，因此它更适合被设计成“个人行动地图册”，而不是“多功能后台”。

### Anchor

选择 `Lo-Fi`，不是继续沿用安全的 `Organic`。  
原因：这个产品不是在卖温和、安静、园艺式效率，而是在管理多条行动线。`Lo-Fi` 更适合做纸张、贴条、旋转、误印、装订、折页、索引卡这些强记忆形态，能把“书本 / 地图 / 沙盘 / 档案”统一到一个体系里。

### Differentiator

整个应用做成“一本正在使用中的行动地图册”：主内容区永远像一本摊开的双页册，页面不是简单切换，而是在切换不同章节和附页。

### 锁定视觉语法

- 主表面：偏饱和纸黄，不用奶油白。
- 版式结构：双页、装订线、索引签、斜贴标签、误位印刷、局部旋转。
- 字体策略：标题与章节名采用混合系统字体碰撞，正文用清晰中文无衬线，数据或短标记用等宽。
- 组件策略：少用“标准管理后台卡片”；多用“地图板 / 索引抽屉 / 档案页 / 航线箱 / 试验托盘 / 设备信号片”。
- 内容纪律：所有文案必须指向真实业务字段，不用空洞口号堆砌。

## 2. 第一轮：主题与板块 Brainstorm

这一轮不急着收敛实现，只先判断“每个页面最该像什么”，并找出可以成立的视觉容器。

### 2.1 总体母题候选

#### 候选 A：行动地图册

- 核心隐喻：一本桌面摊开的行动总册，章节之间切换像翻到不同附页。
- 优点：能覆盖任务、日历、笔记、AI 规划、同步、下载。
- 风险：如果处理太轻，会退化成普通纸感看板。

#### 候选 B：远征指挥箱

- 核心隐喻：一个打开的野外工作箱，内部有抽屉、信号片、工具槽、坐标图。
- 优点：更强烈，适合任务、下载、同步、沙盒。
- 风险：对笔记、设置、工作流会显得过于硬核。

#### 候选 C：侦察档案馆

- 核心隐喻：一组档案夹、缩微卡、索引柜和批注页。
- 优点：适合笔记、AI、工作流、设置。
- 风险：对日历、下载、习惯、沙盒的时间感和操作感不足。

### 2.2 第一轮结论

选 `A 行动地图册` 作为总母题。  
原因：它最容易把“书本、地图、沙盘、档案”合并为同一种世界观，并允许每个页面各自发展出独立的副隐喻。

## 3. 第一轮：按板块发散形态

### 3.1 全局框架

#### Sidebar

候选形式：

- 书脊目录
- 章节切片
- 贴在书页边缘的索引签

判断：

- 最佳形式是“书脊目录 + 索引签”。
- 导航切换不该像 App 菜单，而该像手指拨到某一章节。

#### TopBar

候选形式：

- 页眉注释条
- 装订线上的金属标尺
- 主编批注栏

判断：

- 最佳形式是“页眉注释条”。
- 搜索、创建、AI 入口更像“本页操作工具条”，而不是固定顶栏。

### 3.2 Dashboard

候选形式：

- 今日行动总览双页
- 指挥沙盘
- 值班台

判断：

- 最佳形式是“今日行动总览双页 + 小型沙盘”。
- 左页放今日主线与推进节奏，右页放风险雷达和行动建议。

### 3.3 Tasks

候选形式：

- 周排程折页
- 时间河道图
- 战线甘特布告

判断：

- 周视图最适合“折页时间地图”。
- 全部任务最适合“卡片索引柜”，按状态、优先级、时间进行抽屉化。

### 3.4 Calendar

候选形式：

- 月度地形图
- 坐标挂图
- 行军日历

判断：

- 最佳形式是“月度地形图”。
- 每天不只是日期格，而是一个地块；事件是图钉、旗标、路径线。

### 3.5 Notes

候选形式：

- 情报墙
- 档案盒
- 线索剪报页

判断：

- 最佳形式是“线索剪报页 + 档案抽屉”。
- 笔记要允许标题、标签、关联任务像贴条和批注一样共存。

### 3.6 Habits

候选形式：

- 训练轨迹板
- 养成温室
- 连击章程页

判断：

- 最佳形式是“训练轨迹板”。
- 习惯不是静态指标卡，而是长期刻痕、连击章、周期环。

### 3.7 AiChat

候选形式：

- 参谋手稿台
- 折叠沙盘
- 分镜策划本

判断：

- 最佳形式是“参谋手稿台 + 折叠沙盘”。
- 左侧是对话回合档案，右侧是规划手稿、任务卡、预览图层。

### 3.8 Workflows

候选形式：

- 自动化装配册
- 机关连杆图
- 流程印刷版

判断：

- 最佳形式是“自动化装配册”。
- 每条工作流是一张可翻开的机械装配卡，而不是普通配置项。

### 3.9 Sync

候选形式：

- 信号中继板
- 设备航标图
- 回传收发台

判断：

- 最佳形式是“回传收发台”。
- 设备、队列、同步动作分成三块不同器件，不要还是列表。

### 3.10 Download

候选形式：

- 补给航运单
- 货箱转运架
- 港口调度图

判断：

- 最佳形式是“港口调度图 + 货箱转运架”。
- 下载任务天然适合做成货箱、航线、站点状态。

### 3.11 Sandbox

候选形式：

- 试验托盘
- 工作台切面
- 工具试射场

判断：

- 最佳形式是“试验托盘”。
- 代码、语言、超时、输出像摆在桌面上的器具，不该只是上下两块。

### 3.12 Settings

候选形式：

- 控制面板活页
- 接线图册
- 参数检定簿

判断：

- 最佳形式是“接线图册 + 参数检定簿”。
- AI 配置和通知配置要像两套独立线路，不要在一面普通表单墙上。

## 4. 第二轮：收敛后的统一主题

### 4.1 总主题命名

`Atlas Desk / 行动地图册`

不是做“作战”修辞本身，而是做“桌面上摊开的行动地图册”。  
它包含：

- 章节索引
- 装订线
- 双页布局
- 附签与剪贴
- 折页和插页
- 标注与误印

### 4.2 统一世界观

用户不是在操作一个后台，而是在翻阅自己的行动册：

- 首页是“今日展开页”
- 任务是“排程折页”
- 日历是“月度地图”
- 笔记是“档案剪报”
- AI 是“参谋手稿”
- 工作流是“装配步骤册”
- 同步是“回传台账”
- 下载是“转运页”
- 沙盒是“试验托盘”
- 设置是“接线检定页”

### 4.3 第二轮总差异化动作

做一条贯穿全局的“中缝装订线”。

具体表现：

- 主内容容器始终像摊开的双页，中间有装订缝或压痕。
- 每页默认优先采用左右页结构，而不是均匀卡片网格。
- 某些块允许作为“贴入附页”略微旋转，形成 Lo-Fi 特征。

## 5. 第二轮：设计系统细化

### 5.1 色彩

主色板建议：

- `Paper Base`: `#E8E0C0`
- `Paper Deep`: `#D9CCA6`
- `Paper Shadow`: `#BDA97D`
- `Ink`: `#2D241A`
- `Ink Soft`: `#5C4B38`
- `Signal Red`: `#C74B3A`
- `Survey Blue`: `#4D6A8A`
- `Archive Green`: `#667B49`
- `Tape Cyan`: `#7DB9B6`

使用规则：

- 页面底色保持纸黄。
- 单页内最多 1 个强信号色主导，其他只作辅助。
- 不再用大面积木纹棕色做背景主角，木面可以退到局部纹理或外层桌面。

### 5.2 字体

建议组合：

- 章节标题：`Times New Roman / Georgia / Songti SC`
- 板块标签：`Arial / Helvetica / PingFang SC`
- 数据与短码：`Courier New / SF Mono`

使用规则：

- 章节名允许更大、更挤、更有印刷感。
- 正文必须回归清晰，不走手写体滥用。
- 等宽只用于编号、时间、设备 ID、任务短码、状态片。

### 5.3 结构

统一结构层级：

1. 桌面背景层  
2. 地图册本体层  
3. 页面双页层  
4. 插页 / 贴条 / 卡片层  
5. 注释与工具层

### 5.4 组件语法

组件不再笼统叫 card，建议重新分类：

- `spread`: 双页内容母容器
- `leaf`: 单页页片
- `tab`: 章节索引签
- `clip`: 剪贴块
- `stamp`: 状态章
- `ledger`: 台账条
- `crate`: 货箱块
- `pin`: 地图标记
- `tray`: 试验托盘

### 5.5 动效

动效原则：

- 页切换像翻页或抽出章节，不是普通淡入。
- 局部卡片用轻微旋转复位、滑入、贴附动画。
- AI 规划预览可以像“展开折页”。
- 所有动效控制在 220ms 到 420ms，不做漂浮感。

## 6. 第二轮：页面级落地方案

### 6.1 Sidebar：书脊导航

#### 目标

把侧边栏从“菜单列”改成“章节书脊”。

#### 方案

- 每个导航项做成竖向索引签。
- 当前章节像被抽出一点，露出页边色签。
- `LCC` logo 不再像产品 logo，而像书脊压印。
- 连接状态放到底部小检定签，不做普通状态点。

#### 关键收益

- 全局世界观立即成立。
- 页面切换会有“翻到哪一章”的明确感。

### 6.2 TopBar：页眉注释条

#### 目标

把顶栏从“统一工具条”改成“当前页操作页眉”。

#### 方案

- 左侧显示章节名、页码、当日日期或上下文。
- 搜索框像页眉检索条，不再是一条普通 rounded input。
- `+任务 / +笔记 / AI` 更像插入当前章节的操作章。
- 明暗切换做成“日 / 夜阅读灯”拨片。

### 6.3 Dashboard：今日展开页

#### 目标

首页不是统计板，而是“今天的展开页”。

#### 左页

- 主线任务作为最大标题对象。
- 今日战线卡变成地图上的路线块，不是一组普通卡片。
- 时间、状态、子任务进度压进路线注记里。

#### 右页

- 风险雷达做成批注条和红色标记章。
- AI 建议与快速动作做成贴入页边的便签组。
- 今日进度与 streak 放在页角圆章里。

#### 关键特殊化元素

- 一条“今日主线”粗标线横穿左页。
- 风险项像被红笔圈出。

### 6.4 Tasks：排程折页

#### 周视图

- 采用横向折页时间地图。
- 每天像折页中的一栏地带，不是普通列。
- 任务块像贴在时间河道上的纸条。
- 当前时间线像标尺。

#### 全部任务

- 改成索引抽屉系统。
- 按 `待执行 / 已安排 / 已完成 / 风险` 做抽屉或分页签。
- 选中任务后，右侧出现“任务档案页”，而不是 modal 优先。

#### 关键特殊化元素

- 周视图顶部是折页压痕。
- 任务详情页像抽出的单张档案。

### 6.5 Calendar：月度地图

#### 目标

把月历真正做成地图，而不是“带台词的月历”。

#### 方案

- 每一天是一个地块。
- 事件用图钉、营地旗、路径短线表示。
- 选中某一天时，右侧展开“该地块事件侧页”。
- 月切换像地图平移，而不是普通按钮切页。

#### 关键特殊化元素

- 周末和今天的地块有不同纸质色阶。
- 有事件的日期像被钉过。

### 6.6 Notes：档案剪报

#### 目标

让笔记像有关系、有出处的记录，而不是孤立文本块。

#### 方案

- 笔记列表做成剪报拼贴墙。
- 每条笔记像一张被裁下来的记录页。
- 标签像彩色纸签。
- 与任务关联时，显示连接线或附属条，不只是一个字段。
- 编辑区像“档案补录页”。

#### 关键特殊化元素

- 标题、正文、标签允许形成不完全对齐的排版。
- 选中笔记后，右侧显示“关联任务 / 时间 / 原始记录”。

### 6.7 Habits：训练轨迹板

#### 目标

把习惯从说明型卡片变成“连续行为刻痕”。

#### 方案

- 每个习惯像一条训练轨迹卡。
- 核心视觉不是描述，而是 streak 圆环、频率刻度、最近打卡轨迹。
- 新建习惯区域像“训练项目登记页”。

#### 关键特殊化元素

- 连击数据做成章印或刻度盘。
- 打卡动作像盖章，不像普通按钮反馈。

### 6.8 AiChat：参谋手稿台

#### 目标

保留 `AiChat` 当前已经形成的复杂工作台结构，不做推倒式重构，只把它轻量纳入新的总母体。

#### 总布局

- 左侧：会话档案抽屉
- 中部：对话手稿页
- 右侧：规划沙盘 / 预览折页 / 任务卡托盘

#### 方案重点

- 不改主要信息架构，不重写当前 planning 工具链。
- 只统一表面材质、页边、标题体系和局部容器命名。
- 普通消息仍可保留现有实现，但外观向记录段落和批注条靠拢。
- planning tasks 继续保留索引卡方向，作为全站样式校准参照。

#### 关键特殊化元素

- “当前任务规划”区域继续维持案头感，但不进入大规模结构迁移。
- AI 建议分级时，`must_change` 项像红色批注贴。

### 6.9 Workflows：装配步骤册

#### 目标

让工作流具备机械感和步骤感。

#### 方案

- 每条 workflow 是一张装配卡。
- trigger、action、history 不是平铺字段，而是沿着装配路径排布。
- 历史执行记录像背页翻开后的维修纪录。

#### 关键特殊化元素

- 启用状态像拨杆。
- 执行按钮像“试运行”印章。

### 6.10 Sync：回传收发台

#### 目标

把同步区做成设备和信号的操作面。

#### 方案

- 顶部是收发状态总表盘。
- 中部左侧是同步动作钮组。
- 中部右侧是设备信号片。
- 底部是离线队列回传条。

#### 关键特殊化元素

- 设备不做表格，做“信号片”。
- 队列项像待投递台账条。

### 6.11 Download：港口调度图

#### 目标

让下载任务拥有“运输过程”的空间感。

#### 方案

- 顶部摘要区像港口统计牌。
- 新建下载是“新建航运单”。
- 队列区是“运输货箱架”。
- 每个下载任务是一只货箱，箱面标识状态、速度、进度、分类。

#### 关键特殊化元素

- 进度条不只是一条线，可以做成装载刻度。
- 分类可以像货运标签。

### 6.12 Sandbox：试验托盘

#### 目标

让沙盒像一张正在操作的实验桌。

#### 方案

- 左上是语言与超时配置器件。
- 左下是代码托盘。
- 右侧是输出观察窗。
- 预置模板像不同实验卡。

#### 关键特殊化元素

- 代码区像被压在金属夹板上的试验纸。
- 输出区像示波窗或记录纸。

### 6.13 Settings：接线检定页

#### 目标

让设置不再是一堵表单墙，而是两套系统线路。

#### 方案

- `AI 配置` 和 `通知配置` 各自是一页检定表。
- API Base、Model、Key 像线路端口和参数表。
- Test / Save 是检测动作，不是普通按钮。

#### 关键特殊化元素

- 成功失败反馈要像检定章或测试条。
- 敏感字段允许局部遮罩显示在页侧注记里。

## 8. 实施约束

### 不该做的事

- 不要继续堆“作战 / 战线 / 指挥”等文本来替代真正设计。
- 不要每页仍然回到同一种卡片网格。
- 不要用 emoji 充当主视觉体系。
- 不要为了主题感牺牲真实字段可读性。
- 不要让页面最终变成“很多特效的普通后台”。

### 必须坚持的事

- 一个总主题，多个子容器。
- 真实数据字段始终优先。
- 页面结构先于装饰。
- 同一类信息在全站应有稳定映射。
- AI 页必须成为整个产品的视觉峰值。
