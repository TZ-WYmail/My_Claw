# AI Adviser Book Spread Implementation Plan

## 1. Goal

把当前参谋界面重构成“一本摊开的书”：

- 左页：`回合式顾问对话`
- 右页：`当前战报舱` + `AI 安排任务`
- 书页边缘：`对话档案夹` 作为附着式标签页，而不是孤立侧栏
- 底层视觉：整本书的书脊、纸页、翻页层次、边缘标签、页脚编号统一成立

本次规划优先级：

1. 先恢复稳定显示，不再出现白屏或模块严重变形
2. 再统一双页书本结构
3. 最后细化游戏感、档案感、手账感

新增执行约束：

1. `🤔 反思者质疑` 和 `🎯 规划者视角` 必须落在左侧，不再放在右页
2. `载入今日待办` 必须真正按“今日”语义工作，不能继续等价于“载入所有 pending”
3. 前后端时间基准必须收敛成单一来源，并在前端实时刷新显示

---

## 2. Current Problems

### 2.1 Structural problems

- `回合式顾问对话`
  目前虽然已经做成大页，但仍是“页面中的一个模块”，不是“整本书的左页”。
- `当前战报舱` 和 `AI 安排任务`
  当前还是上下堆叠卡片，更像两个模块，不像书本右页中的不同章节。
- `对话档案夹`
  现在还是独立存在的侧栏，不像附着在书边上的标签系统。

### 2.2 Placement problems

- `推演批注 / 反思者质疑`
  当前挂在右页，和最新交互要求相反。
- `规划者视角`
  当前仍依附在右页预览区，未进入左页的回合阅读链路。
- `Rounds` 下方缺少左侧第二层
  现在左页只有主手稿页，没有“回合之后继续审视与规划”的承接层。

### 2.3 Time problems

- `AiChat` 页面当前用 `new Date()` 本地生成 `currentDisplayTime`
  这只能代表浏览器本机时间，不是仓库内统一时间。
- 后端多个服务仍直接分散调用 `datetime.now()`
  没有统一 helper，也没有统一时区约束。
- `载入今日待办`
  现在实际调用的是 `get_pending_tasks()`，后端返回的是全部 `status = 'pending'` 任务。
  这不是“时间显示错位”的表象问题，而是接口语义本身错误。

### 2.4 Visual problems

- 当前多个模块都保留了独立 `card` 语义，导致界面像“多个盒子排版”，不是一个统一物件。
- 左右内容区边界太硬，缺少书脊、页心、翻页层次、纸张厚度。
- 任务区和战报区没有形成“右页章节”的统一叙事。
- 档案夹和书页主体断裂，视觉上不是同一套东西。

### 2.5 Engineering problems

- 之前已经出现过 `AiChat` 运行时崩溃和 dev/PWA 缓存混淆。
- 当前 `AiChat.jsx` 单文件太大，继续直接堆样式和 JSX 容易再次引发结构性错误。
- 现阶段必须控制修改路径：先稳定，再抽象，再美化。

## 3. Design Direction

### 3.1 Core metaphor

参谋界面不再是“聊天 + 面板”。

它应当是：

- 一本摊开的战略册
- 左页记录问答与推演
- 右页记录结论、排程、风险、任务编成
- 侧边插着多个案卷标签

### 3.2 Target experience

用户进入页面时应感知到：

- 不是在看 SaaS 面板
- 而是在翻一份作战手册 / 战地档案 / 指挥官日记

### 3.3 Style anchors

- 基底：`Lo-Fi paper command board`
- 材质：旧纸页、书脊阴影、边缘标签、页角、夹页、书签
- 信息组织：正文、页边批注、章标题、页脚编号、标签索引
- 游戏感来源：战术册 / 档案册 / 作战日志，而不是浮夸科幻 UI

---

## 4. Implementation Strategy

分 4 个阶段推进。每一阶段都必须可运行、可验收，不允许一次性大改到失控。

在原 4 个阶段之前，新增一个强制纠偏阶段，先把位置与时间语义纠正。

---

## 4.1 Phase 0: Correct Direction First

### 4.1.1 Objective

先修正已经明确跑偏的实现，不继续在错误方向上叠样式。

### 4.1.2 Work

- 左页新增 `left secondary stack`
  - 放 `🤔 反思者质疑`
  - 放 `🎯 规划者视角`
- 右页保留：
  - `当前战报舱`
  - `AI 安排任务`
  - `AI 配置台`
- 后端新增单一时间 helper
  - 统一时区
  - 统一 `now`
  - 统一 `today`
- 前端新增单一 `systemClock` 状态
  - 由后端时间初始化
  - 本地按秒推进显示
  - 只从这一份状态派生 `Rounds` 页头当前时间
- `载入今日待办`
  - 前端明确传 `today_only`
  - 后端仅返回“今日相关任务”
  - 规则：`今日开始`、`今日截止`、`已逾期但仍待办`

### 4.1.3 Deliverable

- `Rounds` 下方左侧出现第二层内容
- 左页阅读链路变成：
  - 回合手稿
  - 反思者质疑
  - 规划者视角
- `载入今日待办` 不再一次性灌入全部 pending
- 左页时间显示来自单一时钟源

### 4.1.4 Acceptance

- 用户进入参谋页后，能在左侧看到 `Rounds` 和其下方的两个附属模块
- 右侧不再承载上述两个模块
- `载入今日待办` 后，任务数量明显小于“全部待办”，且只包含今日相关项
- 左页页头时间每秒刷新，且与后端时间基准一致

## 5. Phase 1: Stabilize Structure

### 5.1 Objective

先把 `AiChat` 的大结构稳定下来，避免继续出现：

- 白屏
- 模块变形
- 高度塌陷
- 容器错位

### 5.2 Work

- 保持当前功能逻辑不动
- 只整理容器层级：
  - `ai-command-shell`
  - `ai-command-main`
  - `ai-command-desk`
  - `viewer modal`
- 去掉目前多余的“独立侧栏心智”
- 确保：
  - 书本主体区域只有一个根容器
  - 左页、右页、标签页都挂在同一书本根容器下
  - 左页内部允许出现 `primary manuscript` + `secondary insight stack` 两层

### 5.3 Deliverable

- 一个稳定的 `book spread shell`
- 不追求精美，只保证结构合理

### 5.4 Acceptance

- 进入参谋界面不白屏
- 界面模块不漂移、不重叠、不只剩背景
- `npm run build` 通过
- `localhost:5173/static/` 打开后刷新不丢结构

---

## 6. Phase 2: Build Book Spread Layout

### 6.1 Objective

把当前三个逻辑块统一进“一本展开的书”。

### 6.2 Layout model

- `ai-book-spread-shell`
  - 书本外层容器
  - 承担透视、阴影、厚度、底座
- `ai-book-spread`
  - 左右双页本体
- `ai-book-page left`
  - 放 `回合式顾问对话`
- `ai-book-page right`
  - 放 `当前战报舱`
  - 放 `AI 安排任务`
  - 放 `AI 配置台`（必要时作为右页底部抽屉）
- `ai-archive-tabs`
  - 贴附在书页左边缘
  - 不再是独立栏

### 6.3 Work

- 去掉左页、右页模块的“独立卡片感”
- 让左页和右页共享同一纸张基底
- 增加：
  - 书脊阴影
  - 页心压痕
  - 左右页轻微不同纸纹
  - 右页比左页略亮或略干净

### 6.4 Deliverable

- 从视觉上能明确看出这是一整本书，而不是多个卡片模块

### 6.5 Acceptance

- 第一眼能看出左页 / 右页 / 书脊
- 标签页像附着在书边上
- 战报和任务区处于同一个右页体系内

---

## 7. Phase 3: Redesign Archive As Attached Tabs

### 7.1 Objective

把“对话档案夹”变成书页边缘的标签系统。

### 7.2 Interaction model

- 默认显示一列标签页耳朵
- 当前选中标签抽出更远
- hover 时轻微抽出
- 标签标题简写显示
- 点击切换当前案卷
- 删除按钮缩成小印章或角标，不破坏整体感

### 7.3 Work

- 不再使用现在的整块 `ai-archive-rail`
- 改为：
  - `ai-archive-tabs`
  - `ai-archive-tab`
  - `ai-archive-tab.active`
- 加入层叠厚度
- 标签颜色分组：
  - 默认纸黄
  - 当前绿色或砖红强调

### 7.4 Deliverable

- 标签页附着在书边，成为书的一部分

### 7.5 Acceptance

- 档案夹不再被感知为独立侧栏
- 选中态清晰
- 在桌面端不抢主内容宽度
- 在移动端可折叠为顶部或侧滑标签列

---

## 8. Phase 4: Refine Page Content Hierarchy

### 8.1 Objective

让左页和右页分别更像“书页内容”，而不是简单堆内容。

### 8.2 Left page

`回合式顾问对话` 改造成：

- 页眉：章节标题、回合编号、日期、页码
- 正文：`任务委托` + `参谋回报`
- 下层：`🤔 反思者质疑`
- 下层：`🎯 规划者视角`
- 附加内容：`推演批注` + `执行动作`
- 页脚：翻页器、当前回合位置

### 8.3 Right page

右页改造成两个章节：

- 上半：`当前战报舱`
  - 情报卡更像战报摘要，不再像零散浮卡
- 下半：`AI 安排任务`
  - 像书中的“战术编成章节”
  - 任务卡保留可编辑能力，但排版更像贴附在页中的索引卡

### 8.4 Work

- 弱化“卡片套卡片”
- 强化章标题、正文、注解、章节分隔
- 给右页加入章节锚点：
  - `战报`
  - `排程`
  - `重排`

### 8.5 Acceptance

- 左页像阅读对话手稿
- 右页像查看战报与作战编成
- 两页信息密度不同，但风格统一

---

## 9. Optional Phase 5: Book-Level Details

这些属于加分项，不应早于前 4 阶段。

### 9.1 Candidate details

- 页角折痕
- 书页编号
- 章节引导线
- 纸边磨损
- 贴纸 / 标签颜色编码
- 书脊压纹
- 翻页轻动效
- 标签抽出缓动
- 当前页微微抬起阴影
- 任务确认时右页章节点亮

### 9.2 Rule

只在不影响可读性、不影响性能的前提下加。

## 10. Refactor Plan

当前 `AiChat.jsx` 过大，建议在阶段 2 完成后开始拆分。

### 10.1 Target split

- `AiChat.jsx`
  - 只保留状态和顶层组装
- `components/ai-book/ArchiveTabs.jsx`
- `components/ai-book/ConversationPage.jsx`
- `components/ai-book/ResultPage.jsx`
- `components/ai-book/PlanningPageSection.jsx`

### 10.2 Rule

- 先完成稳定的双页样式再拆
- 拆分时不改变行为

---

## 11. Risk Control

### 11.1 Main risks

- JSX 层级调整过大导致再次白屏
- 复杂样式导致左右页高度错位
- `planning` 区块太重，塞进右页后可读性下降
- 标签页吸附后移动端布局失控

### 11.2 Mitigation

- 每阶段只改一层：
  - 先容器
  - 再书本
  - 再标签
  - 再细节
- 每阶段都执行：
  - `npm run build`
  - 打开参谋页检查
- 一旦再次出现白屏：
  - 回退到上一个结构稳定点
  - 不继续堆视觉细节

---

## 12. Immediate Next Step

下一步不直接追求最终视觉，而是先完成：

1. 新增后端统一时间 helper 与时间接口
2. 把前端 `Rounds` 当前时间改为单一 `systemClock`
3. 把 `载入今日待办` 改成真正的 `today_only`
4. 把 `反思者质疑 / 规划者视角` 移到左页 `Rounds` 下方
5. 再继续推进书签、木纹、流式柔化与整本书样式收束
4. 保持逻辑不变，只做结构和样式映射

这一步完成后，再进入细化阶段。

---

## 13. Done Definition

当满足以下条件时，参谋台重构视为第一轮完成：

- 用户进入参谋页时，不再觉得是在看几个模块
- 用户第一眼能理解为“一本展开的战略册”
- 左页是顾问对话，右页是战报与任务
- 档案夹是附着在书边上的标签，而不是孤立栏
- 所有现有功能仍可用
- 无白屏、无严重变形、无旧缓存干扰
