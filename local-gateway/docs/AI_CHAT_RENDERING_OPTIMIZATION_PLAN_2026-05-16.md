# AI 对话渲染与交互优化执行文档

日期：2026-05-16  
范围：`local-gateway/frontend/src/pages/AiChat.jsx` 及相关前端渲染链路

## 1. 当前现状

当前 AI 对话页已经具备以下能力：

1. assistant 正文支持 Markdown 渲染
2. thinking 区域与正文区分展示
3. 代码块支持复制、展开
4. 表格支持展开
5. Mermaid 代码块支持动态真图渲染
6. Mermaid 渲染具备基本防抖与重复渲染抑制
7. PWA 预缓存范围已初步收缩

但仍然存在以下工程问题：

1. `AiChat.jsx` 文件体积过大，职责过多
2. Markdown 解析、代码块交互、Mermaid 渲染、消息流式更新耦合在一个文件中
3. assistant 正文仍然是“整块字符串 → 整块 HTML”的渲染路径
4. 长消息在流式输出下仍可能有整段重排成本
5. Mermaid 渲染逻辑仍嵌入主页面，后续维护成本高
6. 代码高亮仍为轻量规则，结构不可扩展
7. 缺少专门的 UI 层测试与渲染基准约束

## 2. 优化目标

目标不是简单“功能更多”，而是让 AI 对话部分具备以下特征：

1. **职责拆分清晰**：页面容器、消息列表、消息气泡、Markdown 渲染、Mermaid 渲染分层
2. **动态渲染稳定**：流式输出时只更新必要节点
3. **渲染性能可控**：减少长消息和图表带来的重复渲染
4. **结构可维护**：后续继续加表格能力、代码高亮、公式支持时不需要反复堆在 `AiChat.jsx`
5. **体验可扩展**：为后续继续加 Mermaid 缩放、图表导出、代码语言增强留接口

## 3. 总体分期

### Phase 1：组件拆分与渲染解耦

目标：

1. 把 `AiChat.jsx` 中与消息正文渲染相关的逻辑抽离
2. 建立独立组件：
   - `ChatMessageBubble`
   - `AssistantMarkdown`
   - `MermaidBlock`
   - `CodeBlock`
3. 保持现有行为不退化

交付标准：

1. `AiChat.jsx` 只保留页面编排、状态和事件
2. Mermaid 与 Markdown 渲染逻辑不再直接混在主页面

---

### Phase 2：流式增量渲染优化

目标：

1. assistant 长消息在流式输出中减少整块重新计算
2. Markdown 渲染增加消息级 memo 与段落级拆分基础
3. Mermaid 渲染仅在代码块闭合后触发

交付标准：

1. 流式输出时 CPU 抖动明显下降
2. Mermaid 不在半截代码块阶段反复尝试渲染

---

### Phase 3：可视交互增强

目标：

1. Mermaid 图支持更好的查看体验
2. 代码块查看器支持更稳定的全屏阅读
3. 大表格支持单独阅读容器

交付标准：

1. 图表、代码、表格都具备独立查看体验
2. 页面主布局不因大块内容被破坏

---

### Phase 4：可维护性与测试补强

目标：

1. 给渲染器和组件补单测/快照测试
2. 建立最小渲染契约，避免后续改坏

交付标准：

1. Markdown 转换核心逻辑至少有基础测试覆盖
2. Mermaid/代码块/表格关键交互有回归保护
