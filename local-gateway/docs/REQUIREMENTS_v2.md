# LocalCommandCenter 用户需求文档 v2

> 基于用户真实使用反馈，持续迭代的补充需求

## 一、新需求分析

### 需求 A：周日历增加时间纵轴

**问题**：当前周日历只有 7 列平铺任务卡片，用户无法直观看到任务在不同时间段内的分布。缺少类似 Google Calendar / Outlook 的时间轴视图。

**期望效果**：
- 日历左侧显示时间纵轴（07:00 ~ 23:00，每小时一行）
- 任务按实际时间定位到对应的时间格中
- 未安排任务的时间段保持空白
- 可滚动查看一天中的时间段

### 需求 B：AI 对话控制面板

**问题**：目前每个操作都需要手动点击不同标签页、填写表单。用户希望能像用 GLM 智能体一样，通过自然语言对话直接操控后端全部功能。

**期望效果**：
- 右下角浮动 AI 对话入口按钮（类似客服聊天气泡）
- 点击展开对话面板，支持文字输入
- 后端接收用户消息后，转发给配置的 AI API（如 GLM）
- AI 理解意图后自动调用本地网关的 5 个工具 API
- 实时显示 AI 的思考过程和调用结果
- 对话历史在本地保留

**技术方案**：
- 后端新增 `/api/chat` 端点，接收用户消息
- 后端封装 AI API 调用（支持 OpenAI/GLM 兼容格式）
- AI 携带 5 个工具的 Schema 作为 function calling
- 后端执行 AI 返回的 tool call，将结果返回给 AI 继续对话
- 前端新增对话悬浮窗 UI

## 二、实现计划

### 阶段 1（本次实现）

1. ✅ 日历改为带时间纵轴的周日历（07:00-23:00）

3. ✅ 前端 AI 对话悬浮窗（右下角气泡 + 对话面板）
4. ✅ AI 可操控全部 5 个工具（任务/下载/搜索/沙盒/异步查询）
