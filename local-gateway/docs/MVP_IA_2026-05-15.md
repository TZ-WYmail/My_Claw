> 文档版本: 1.0
> 更新日期: 2026-05-15
> 依据文档: `MVP_PRD_2026-05-15.md`

---

## 一、目标

本信息架构文档用于定义 MVP 版本的页面层级、导航关系、主路径和低频功能收口方式。

目标只有一个：

**让用户在最少切换下完成“看今天 -> 排任务 -> 做事情 -> 记内容 -> 看结果”的闭环。**

---

## 二、一级导航

MVP 一级导航固定为 6 个入口：

1. `今天`
2. `任务`
3. `日历`
4. `笔记`
5. `AI 助手`
6. `设置`

原则：

- 一级导航只保留高频主路径
- 不把技术能力和低频模块放进一级导航
- 页面名称使用用户语言，不使用实现语言

---

## 三、导航层级

### 3.1 主导航

#### `今天`

- 今日重点任务
- 今日日程摘要
- 逾期提醒
- 最近内容
- 快速入口

#### `任务`

- 今日视图
- 周视图
- 任务详情
- 创建/编辑任务

#### `日历`

- 周视图
- 月视图
- 时间块
- 日程详情

#### `笔记`

- 最近笔记
- 所有笔记
- 编辑/预览
- 模板创建

#### `AI 助手`

- 安排今天
- 拆解任务
- 整理笔记
- 日终回顾
- 对话视图

#### `设置`

- 外观
- AI 基础配置
- 系统状态

---

### 3.2 次级入口

以下内容不放入主导航，但可作为次级入口存在：

1. 习惯卡片
   - 放在 `今天` 页

2. 实验功能区
   - 下载
   - 工作流
   - 同步
   - 沙盒

3. 全局搜索
   - 顶栏统一入口

4. 快速创建
   - 顶栏或浮动入口

## 五、主用户路径

### 路径 1：开始一天

`今天` -> 看今日重点 -> 打开任务 -> 调整时间 -> 回到今天页确认

### 路径 2：执行任务

`今天` -> 点击任务 -> `任务` -> 开始执行 -> 创建/打开关联笔记 -> `笔记`

### 路径 3：安排本周

`任务` -> `周视图` -> `日历` -> 调整时间 -> 返回 `任务`

### 路径 4：借助 AI

`今天` 或 `任务` -> `AI 助手` -> 选择“安排今天/拆解任务” -> 生成建议 -> 确认 -> 回填到任务或笔记

### 路径 5：结束一天

`今天` -> 查看完成情况 -> `AI 助手` -> 日终回顾 -> 生成总结 -> 写入 `笔记`

## 六、页面职责边界

### `今天`

负责：

- 决策
- 提醒
- 快速进入下一步

不负责：

- 复杂编辑
- 全量数据浏览
- 技术配置

### `任务`

负责：

- 任务本身的创建、修改、完成
- 今日和本周执行管理

不负责：

- 长篇记录
- 复杂自动化

### `日历`

负责：

- 时间安排
- 冲突发现
- 周/月级视角

不负责：

- 任务详情深度编辑
- 项目管理

### `笔记`

负责：

- 记录
- 整理
- 回看

不负责：

- 复杂任务调度
- 自动化配置

### `AI 助手`

负责：

- 提供建议
- 帮助整理
- 降低思考和组织成本

不负责：

- 接管整个系统
- 黑箱执行高风险能力

### `设置`

负责：

- 必要配置
- 系统状态说明

不负责：

- 承载产品主功能

## 七、低频功能收口

MVP 中，以下功能统一放入“实验功能”或“高级功能”区：

1. 下载
2. 工作流
3. 同步
4. 沙盒

收口原则：

- 不在一级导航展示
- 不在首页主路径展示
- 不使用与主功能同等视觉权重

---

## 八、顶栏结构建议

顶栏只保留 4 类高频入口：

1. 全局搜索
2. 新建任务
3. 新建笔记
4. AI 快速入口

不建议在顶栏放：

1. 低频实验功能
2. 大量状态展示
3. 技术参数控制

---

## 九、移动与窄屏原则

虽然 MVP 主面向桌面，但信息架构应满足窄屏场景：

1. 一级导航可折叠为底部导航或抽屉
2. 今天页卡片按优先级顺序线性展开
3. 任务与笔记应支持单栏模式

---

## 十、总结

MVP 信息架构的重点不是“把所有功能都挂上去”，而是：

1. 一级导航只保留高频主路径
2. 每个页面只承担一种清晰职责
3. 用户在首页就能进入下一步动作
4. 低频复杂能力必须收口

只有这样，产品才会从“模块集合”变成“有主线的系统”。
