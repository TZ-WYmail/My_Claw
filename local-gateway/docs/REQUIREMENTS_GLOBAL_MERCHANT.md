# 全球贸易商人 AI 日程助手 — 需求规格文档

> **版本**: v1.0
> **日期**: 2026-04-28
> **目标用户**: 从事国际贸易的商人（跨时区、跨文化、高频决策、多线程并行）
> **产品定位**: 内置 AI 的智能日程调度系统，随时随地动态规划时间与任务

---

## 一、用户画像

### 1.1 核心用户：全球贸易商人

**典型特征**：
- 业务横跨 3-8 个时区，与供应商（中国/东南亚）、客户（欧美/中东）、物流（全球）频繁交互
- 每日处理 20-50 条待办事项，其中 30% 有严格时间窗口（报关截止、船期截单、信用证到期）
- 决策链路短但容错低——错过一个船期意味着延误 2-4 周，错过信用证有效期意味着货款冻结
- 出差频繁，一个月内可能经历 3-5 次时区切换（上海→迪拜→伦敦→纽约）
- 信息源分散：邮件、微信/WhatsApp、贸易平台、海关系统、银行通知、行业新闻

**关键痛点**：
| # | 痛点 | 现状 | 期望 |
|---|------|------|------|
| P1 | 时区混乱 | 手动换算，经常搞错会议时间 | 自动识别参与者时区，智能推荐最佳时段 |
| P2 | 任务优先级漂移 | 紧急事项淹没重要事项 | AI 动态评估优先级，实时重排日程 |
| P3 | 信息孤岛 | 报关单在邮件、合同在网盘、日程在日历 | 统一入口，AI 关联上下文 |
| P4 | 倒时差效率断崖 | 跨时差后认知下降，日程照搬导致低效 | AI 根据生物节律自适应调整日程密度 |
| P5 | 突发事件打断 | 船期变动、政策变化、汇率波动随时打断计划 | AI 实时感知并动态重塑日程 |
| P6 | 移动端依赖 | 出差途中 80% 操作在手机完成 | 原生级移动体验，离线可用 |

### 1.2 衍生用户

- **贸易公司助理**：需要代老板查看/调整日程，管理会议邀约
- **物流协调员**：需要查看货主可用时间窗口，安排验货/装柜
- **外贸 SOHO**：一人身兼数职，对自动化依赖极强

## 二、核心场景用例

### 场景 1：晨间动态日程生成

> **时间**：上海 07:00（刚从纽约飞回，-12h 时差）
> **触发**：用户打开助手

**系统行为**：
1. 感知用户当前时区（上海，UTC+8）和最近时差变化（从 UTC-5 切换）
2. 识别"时差恢复模式"——今日建议负荷降低 40%
3. 扫描今日待办：
   - 09:00 原定客户视频会议（伦敦，UTC+1）→ 提示"对方下午 2 点，你刚倒时差，建议改期或缩短至 15 分钟"
   - 11:00 报关截止（不可移动）→ 标记为红色优先，前置提醒 30 分钟
   - 14:00 供应商验厂跟进 → 建议"延至 16:00，午休后精力更佳"
   - 一封信用证修改通知（今晨邮件）→ AI 提取关键信息，自动创建"确认信用证修改"任务，截止时间 = 48h 后
4. 输出一份 **动态日程表**，按 30 分钟粒度排列，包含精力曲线标注：

```
07:00-07:30 ☀️ 缓醒模式 — 饮水、简单拉伸
07:30-08:00 📧 快速扫描关键邮件（AI 已预筛选 3 封需立即处理）
08:00-08:30 🔴 报关截止前置准备 — 核对单据清单
08:30-09:00 ⚡ 完成报关提交（截止 09:00）
09:00-09:15 ☕ 短会：伦敦客户（已压缩至 15 分钟）
09:15-10:30 🧘 恢复期 — 无排程，允许弹性处理积压邮件
10:30-11:30 📊 信用证修改审核 + 回复银行
11:30-12:00 🍽️ 午餐
12:00-14:00 💤 建议午休（时差恢复）
14:00-14:30 📋 查看今日物流追踪（AI 已汇总 5 批货物状态）
14:30-16:00 🏭 供应商验厂视频跟进
16:00-17:00 📝 明日预备：迪拜客户来访准备
```

### 场景 2：突发事件动态重塑

> **时间**：上海 14:00
> **事件**：收到通知——原定明日出发的船期提前至今日 18:00 截单

**系统行为**：
1. 即时推送高优先级通知："船期截单提前 18h，影响 3 项待办"
2. AI 分析影响面：
   - 15:00 原定内部会议 → 建议改期或改为异步沟通
   - 装柜清单尚未最终确认 → 标记为紧急
   - 报关预录入需要 2 小时 → 必须现在开始
3. 自动生成 **重塑日程**：
   - 14:00-14:30 完成装柜清单确认
   - 14:30-16:30 报关预录入
   - 16:30-17:00 联系货代确认截单细节
   - 17:00-17:30 缓冲时间
   - 15:00 的会议 → 移至明日 10:00，已自动发送改期建议
4. 用户确认后，系统自动执行：修改日程、发送改期通知、更新任务截止时间

### 场景 3：跨时区会议智能排期

> **需求**：安排一次上海、迪拜、伦敦三方的视频会议

**系统行为**：
1. 获取三方的工作时间偏好：
   - 上海方：09:00-18:00 UTC+8
   - 迪拜方：09:00-17:00 UTC+4
   - 伦敦方：09:00-17:00 UTC+1
2. 计算时区交集窗口：17:00-18:00 上海时间（即 13:00-14:00 迪拜，09:00-10:00 伦敦）
3. 检查三方日历已有占位，推荐 2-3 个可用时段
4. AI 建议："周三 17:30 上海时间对各方最优，伦敦方刚上班精力充沛，迪拜方午休后，上海方虽在下班前但可接受"
5. 用户确认后，自动创建三时区标注的日历事件，并生成对应时区的邀约消息模板

### 场景 4：出差行程连贯规划

> **需求**：5 天出差——上海→曼谷→迪拜→回上海

**系统行为**：
1. 用户语音输入："下周去曼谷和迪拜出差，周一出发周四回"
2. AI 自动：
   - 创建出差框架任务（航班、酒店、签证检查）
   - 识别时区变化：上海(+8)→曼谷(+7)→迪拜(+4)→上海(+8)
   - 在每个城市生成适配当地时间作息的日程模板
   - 曼谷段：安排供应商拜访、工厂验货
   - 迪拜段：安排客户晚宴（注意当地晚间商务文化）
   - 回程日：降低负荷，预留恢复期
3. 检测冲突：周三迪拜的会议与国内周一创建的周四截止任务冲突 → 提前提醒并建议委托处理

### 场景 5：语音快速捕获

> **场景**：开车途中接到供应商电话，对方变更了交货日期

**系统行为**：
1. 用户长按手机说："供应商鑫达塑胶，交货从15号改到20号，需要更新PO-2026-0312"
2. 语音识别 + AI 语义解析：
   - 提取实体：供应商=鑫达塑胶，原日期=15号，新日期=20号，订单号=PO-2026-0312
   - 自动匹配已有任务："鑫达塑胶交货跟进"
   - 更新截止时间为 20 号
   - 创建备忘笔记："供应商电话确认交货延期至20号，原因待确认"
3. 推送确认："已更新鑫达塑胶交货日期至 20 号，是否需要通知物流方？"

### 场景 6：周期性贸易任务管理

> **场景**：每月重复的贸易操作——信用证跟踪、船期确认、报关、对账

**系统行为**：
1. 支持"贸易周期模板"：定义一套周期任务链
   - T-30：确认订单 → T-21：备货检查 → T-14：订舱 → T-7：报关准备 → T-3：装柜 → T-0：截单 → T+3：提单确认 → T+7：付款
2. 每创建新订单，自动展开这条任务链到日历
3. 任一环节延误，自动后推下游任务并通知相关人员
4. AI 每日晨报汇总：本周 3 批货物所处环节、下周 2 批需提前准备

### 场景 7：数据驱动决策辅助

> **场景**：汇率波动影响利润空间

**系统行为**：
1. Webhook 接收汇率预警（如 USD/CNY 突破 7.35）
2. AI 计算影响：当前未结算订单 3 笔，预计损失 ¥12,500
3. 自动创建紧急任务："评估是否提前锁汇"
4. 在日程中插入今日 30 分钟分析时段
5. 推送简报："USD/CNY 7.35，影响 3 笔未结订单，建议今日内决策是否锁汇"

## 三、功能需求规格

### 3.1 时区智能引擎（核心差异化）

#### FR-TZ-01 多时区感知与自动切换
- **描述**：系统自动检测用户当前时区，所有时间显示自动转换
- **输入**：设备 GPS / 网络位置 / 手动选择
- **输出**：统一显示为用户当前本地时间，同时标注原始时区
- **规则**：
  - 日历视图同时显示本地时间和原事件时区
  - 任务截止时间始终以"用户当前时区"为基准展示
  - 时区变更时（跨越时区），触发日程重评估

#### FR-TZ-02 跨时区会议窗口计算
- **描述**：给定 N 个参与者时区，自动计算可用交集时间窗口
- **算法**：
  1. 每个参与者定义"工作时间窗口"（可自定义，默认 09:00-18:00 本地时间）
  2. 转换为 UTC 基准计算交集
  3. 评估每个窗口的"舒适度分数"（考虑：是否在用餐时间、是否在精力低谷、是否在非工作时间边缘）
  4. 按舒适度排序推荐 Top 3 时段
- **边界**：支持最多 10 个时区的窗口计算

#### FR-TZ-03 时差恢复模式
- **描述**：检测跨越 ≥3 个时区的旅行后，自动启用恢复模式
- **规则**：
  - 跨 3-5 个时区：Day 1 负荷降至 60%，Day 2 降至 80%，Day 3 恢复
  - 跨 6+ 个时区：Day 1 负荷降至 40%，Day 2 降至 60%，Day 3 降至 80%，Day 4 恢复
  - 恢复期间：避免安排在精力低谷时段的高认知任务
  - 用户可手动关闭恢复模式
- **输入**：航班信息 / 日历出差标记 / GPS 时区变化

#### FR-TZ-04 时区标注系统
- **描述**：所有时间相关实体支持多时区标注
- **格式**：`2026-04-28 14:00 (UTC+8) / 2026-04-28 06:00 (UTC+0) / 2026-04-28 01:00 (UTC-5)`
- **规则**：日历事件至少显示用户当前时区 + 事件原始时区；涉及他人时额外标注对方时区

### 3.2 AI 动态日程引擎（核心差异化）

#### FR-DS-01 晨间动态日程生成
- **描述**：每日首次打开系统时，AI 生成当日动态日程
- **流程**：
  1. 扫描今日所有待办任务、会议、截止事项
  2. 评估用户状态（时差、近期工作负荷、健康数据可选）
  3. 按优先级和精力曲线排列任务
  4. 输出 30 分钟粒度的日程表，含精力标注
- **触发**：每日首次访问 / 手动触发 / 07:00 自动推送
- **约束**：生成时间 ≤ 3 秒

#### FR-DS-02 实时优先级重评估
- **描述**：当外部事件到达时，AI 实时重评估所有待办优先级
- **触发条件**：
  - 新任务创建
  - 任务截止时间变更
  - 外部事件到达（Webhook / 邮件标记 / 手动标记紧急）
  - 船期/航班变更
- **算法**：基于 Eisenhower 矩阵 + 贸易业务权重 + 时效性衰减函数
- **输出**：重排后的日程 + 变更说明（"以下任务优先级已调整，原因：…"）

#### FR-DS-03 日程冲突智能化解
- **描述**：自动检测日程冲突并给出解决方案
- **冲突类型**：
  - 时间重叠：两个事件占据同一时段
  - 精力冲突：高认知任务排在精力低谷
  - 依赖冲突：前置任务未完成但后续已排
- **解决策略**：
  - 自动方案：移动低优先级事件、拆分长任务、合并相关会议
  - 建议方案：列出 2-3 个选项供用户选择
  - 紧急方案：冲突涉及不可移动截止时间时，直接排入日程并通知用户

#### FR-DS-04 精力感知排程
- **描述**：根据用户精力曲线分配不同类型任务
- **精力模型**：
  - 09:00-11:00 高精力 → 决策、谈判、重要邮件
  - 11:00-12:00 中精力 → 常规沟通、文档审核
  - 14:00-15:00 中精力 → 创意工作、方案设计
  - 15:00-17:00 低精力 → 行政事务、数据整理、简单回复
  - 可自定义精力曲线
- **时差影响**：跨时差后精力曲线平移 + 幅度降低

#### FR-DS-05 自适应日程密度
- **描述**：根据当日任务量和用户状态自动调整日程密度
- **规则**：
  - 高负荷日（>8h 任务）：紧凑排列，15 分钟缓冲
  - 正常日（4-8h）：标准排列，30 分钟缓冲
  - 低负荷日（<4h）：宽松排列，60 分钟缓冲 + 深度工作时间块
  - 时差恢复日：强制 30% 空档，不连续排满超过 2 小时

### 3.3 贸易业务专属功能

#### FR-TR-01 贸易任务链模板
- **描述**：支持定义可复用的贸易流程任务链
- **模板结构**：
  ```
  模板名称: FOB出口标准流程
  任务链:
    - T-30: 确认订单细节 → 优先级:高 → 负责人:业务员
    - T-21: 备货完成确认 → 优先级:高 → 负责人:跟单员
    - T-14: 订舱确认 → 优先级:高 → 依赖:备货完成
    - T-7:  报关资料准备 → 优先级:高 → 依赖:订舱确认
    - T-3:  装柜安排 → 优先级:中 → 依赖:报关准备
    - T-0:  截单 → 优先级:紧急 → 依赖:装柜
    - T+3:  提单确认 → 优先级:高
    - T+7:  尾款支付 → 优先级:高 → 依赖:提单确认
  ```
- **实例化**：输入基准日期（如船期），自动展开所有任务到日历
- **级联更新**：任一任务延期，自动重算下游所有任务日期

#### FR-TR-02 关键日期看板
- **描述**：汇总所有贸易关键截止日期的可视化看板
- **维度**：
  - 本周截止：报关截单、信用证到期、付款截止
  - 在途追踪：每批货物的 ETA、当前位置
  - 即将到期：未来 7/14/30 天内的关键节点
- **交互**：点击任意节点展开详情、跳转关联任务

#### FR-TR-03 上下文关联
- **描述**：任务、文件、日程、笔记之间建立业务上下文关联
- **关联模型**：
  - 订单号 → 关联：合同文件、报关单、付款记录、物流追踪、邮件往来
  - 供应商 → 关联：所有历史订单、质量记录、沟通笔记
  - 客户 → 关联：所有报价记录、合同、售后服务记录
- **AI 能力**：用户问"PO-2026-0312 的进度"，AI 自动汇总关联链上的所有状态

#### FR-TR-04 多币种与汇率追踪
- **描述**：在任务和财务相关项目中支持多币种标注和汇率换算
- **功能**：
  - 任务可标注金额（含币种），如"支付尾款 USD 25,000"
  - 自动换算为用户本币显示
  - 汇率变动超阈值时触发预警任务
- **数据源**：通过 Webhook 接入外部汇率 API

#### FR-TR-05 合规日历
- **描述**：内置贸易合规关键日期提醒
- **内容**：
  - 各国节假日（影响物流和银行操作）
  - 关税政策变更日期
  - 许可证/资质到期日
  - 反倾销调查期限
- **AI 能力**：自动检测新创建的贸易任务是否涉及即将到来的合规风险

### 3.4 语音交互系统

#### FR-VO-01 多语言语音识别
- **描述**：支持中英混合语音输入（贸易场景核心需求）
- **语言**：中文（普通话）、英语、阿拉伯语、日语
- **处理**：自动检测语言，支持语种切换和混合识别
- **精度**：贸易专业术语识别准确率 ≥ 90%（FOB、CIF、L/C、B/L 等）

#### FR-VO-02 语音快速创建
- **描述**：一句话创建复杂任务
- **示例输入**："下周一前确认和迪拜 Al Fahim 的合同，优先级高，关联 PO-2026-0415"
- **AI 解析输出**：
  - 任务名：确认 Al Fahim 合同
  - 截止：下周一（自动计算日期）
  - 优先级：高
  - 关联：PO-2026-0415
  - 提示：迪拜下周一是当地工作日，建议提前到周日确认

#### FR-VO-03 语音查询
- **描述**：语音查询任务状态和日程
- **示例**："今天下午有什么安排？"、"信用证那个任务什么状态？"、"下周去曼谷的出差准备得怎么样了？"
- **输出**：语音播报 + 屏幕显示

#### FR-VO-04 离线语音
- **描述**：在网络差的环境（如飞机上、偏远工厂）仍可语音创建任务
- **机制**：本地 Whisper 模型做初步识别，联网后 AI 补全语义解析

### 3.5 通知与预警系统

#### FR-NT-01 分级通知
- **描述**：根据紧急程度分级推送通知
- **级别**：
  - 🔴 紧急：船期变更、信用证到期（24h 内）、报关截止 → 强推送 + 声音 + 震动
  - 🟡 重要：会议提醒（15 分钟前）、任务即将到期（24h） → 标准推送
  - 🟢 信息：每日日程摘要、习惯提醒 → 静默推送 / 角标
- **免打扰**：支持按时段设置免打扰（如 22:00-07:00 本地时间，紧急除外）

#### FR-NT-02 智能提醒时机
- **描述**：根据任务类型和用户状态选择最佳提醒时机
- **规则**：
  - 报关截止：提前 2h + 1h + 30min 三次提醒
  - 会议：提前 15min，跨时区会议额外提前 5min（含时区确认）
  - 船期相关：截单前 24h、12h、6h、1h 四次提醒
  - 时差恢复期：不推送非紧急通知

#### FR-NT-03 预警触发器
- **描述**：基于条件的主动预警
- **触发条件**：
  - 汇率波动超过设定阈值
  - 货物物流状态异常（滞留、退回）
  - 合规政策变更（通过 Webhook 或 RSS 检测）
  - 重复任务连续未完成 ≥ 3 次
- **动作**：创建预警任务 + 推送通知 + 附加 AI 分析建议

### 3.7 AI 对话与规划

#### FR-AI-01 业务语义理解
- **描述**：AI 理解贸易领域自然语言指令
- **能力**：
  - 解析贸易术语（FOB、CIF、EXW、L/C、B/L、D/P 等）
  - 识别常见缩写（PO、PI、S/C、ETD、ETA 等）
  - 理解时间表达（"截单前"、"装柜当天"、"开船后第三天"）
  - 上下文理解（"那个迪拜的单子"→ 最近关联迪拜的订单）

#### FR-AI-02 主动建议
- **描述**：AI 在适当时机主动提供建议
- **场景**：
  - 检测到连续 3 天未处理某个高优先级任务 → "这批货的报关资料准备进度如何？截止还剩 2 天"
  - 检测到今天日程过满 → "今日 10 项任务，建议将 3 项非紧急项移至明日"
  - 检测到任务链前置未完成 → "装柜任务明日截止，但报关准备尚未完成，需要协助？"
  - 周五下午 → 生成本周总结 + 下周预览

#### FR-AI-03 智能任务拆解
- **描述**：AI 将复杂任务自动拆解为子任务
- **示例**：
  - 输入："完成迪拜客户的报价"
  - 拆解：
    1. 核实产品规格和数量
    2. 计算成本（含汇率换算）
    3. 确认物流方案和运费
    4. 制作 Proforma Invoice
    5. 发送报价邮件
  - 每个子任务自动分配预估时间

#### FR-AI-04 上下文记忆
- **描述**：AI 记住业务上下文，跨对话保持一致
- **范围**：
  - 活跃订单及其状态
  - 常用供应商/客户信息
  - 用户习惯和偏好
  - 近期决策和原因
- **遗忘机制**：超过 90 天未引用的上下文降权

### 关键技术挑战

1. **时区引擎**：需要在数据层（SQLite 存储 UTC）、服务层（业务逻辑感知时区）、展示层（多时区标注）三层全面改造
2. **AI 日程引擎**：需要设计优先级评估算法、精力曲线模型、冲突消解策略，当前 ai_service 的 function calling 框架可以扩展
3. **任务链级联**：当前 task_service 的子任务是静态关联，需要改为动态依赖图 + 级联传播
4. **离线 AI**：需要本地模型（如 Ollama + Qwen）做降级方案，当前依赖远程 API
